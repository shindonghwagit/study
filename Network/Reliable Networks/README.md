## 1. Network Architecture (네트워크 아키텍처)

> 네트워크 아키텍처란 **네트워크를 통해 데이터를 이동시키는 인프라(나 네트워크)를 지원하는 기술들**을 의미한다.

사용자의 기대를 충족하기 위해 네트워크 아키텍처가 반드시 갖춰야 할 **4가지 핵심 특성**:

| 특성 | 한국어 | 핵심 요약 |
|------|--------|-----------|
| **Fault Tolerance** | 결함 허용성 | 장애가 발생해도 네트워크가 계속 동작 |
| **Scalability** | 확장성 | 사용자/장치가 늘어나도 유연하게 확장 가능 |
| **Quality of Service (QoS)** | 서비스 품질 | 흐름을 트래픽을 우선순위에 따라 처리 |
| **Security** | 보안 | 데이터와 네트워크를 위협으로부터 보호 |

### QoS 관련 지연시간(Latency) 기준

```
Real-time (RT)         →  10ms 이내    ex) 게임, 주식 자동매매
TeleConferencing       →  100ms ~ 150ms (음성/영상 싱크 맞추기 위함)
Streaming (video/voice)→  100ms ~ 170ms
```

> - **QoE (Quality of Experience)** 도 함께 알아두자!
> - **QoS**: 네트워크 장비 입장의 객관적 수치 (대역폭, 지연시간 등)
> - **QoE**: 사용자가 체감하는 주관적 만족도
> - PS-LTE 예시: Public Safety(공공안전) 전용망 → 재난 문자, 독립망 등

## 2. Fault Tolerance (결함 허용성)

> **장애가 발생했을 때 영향받는 장치의 수를 최소화**하여 장애의 영향을 제한하는 것

- 장애에 의한 영향을 최소화해서 장애 영향을 제한
- 이를 위해 **여러 개의 경로(Multiple paths)** 가 반드시 필요

### 구현 방법 → Packet Switched Network

| 개념 | 설명 |
|------|------|
| **Packet Switching** | 트래픽을 **패킷 단위로 쪼개서** 전송 → 패킷마다 다른 경로 이용 가능 |
| **Redundant Connections** | 장치나 링크 장애 시 **대체 경로(대비 경로)** 로 자동 우회 |

### Packet Switching vs Circuit Switching

| 구분 | Packet Switching | Circuit Switching |
|------|-----------------|-------------------|
| **방식** | 패킷 단위로 쪼개서 전송 | 전용 회선(circuit)을 미리 확보 |
| **경로** | 패킷마다 다른 경로 가능 | 고정된 하나의 경로만 사용 |
| **장애 시** | 다른 경로로 우회 가능 | 회선이 끊기면 통신 불가 |
| **예시** | 인터넷 | 옛날 유선 전화 |

```
Packet Switched를 쓰므로:
  패킷 단위로 쪼개서 전달하기 때문이
  패킷마다 다른 경로를 이용할 수 있어,
  고장에 대비해 영향을 최소화하는 것이 가능 → 우리가 쓰는 인터넷!

Circuit Switched는 전화선(circuit)을
미리 할당하기 때문에 연결이 고정된 채로 사용
→ Circuit에서 회선이 끊기면 통신 불가 ex) 옛날 유선 전화
```

## 3. Scalability (확장성)

> **기존 사용자의 서비스 품질에 영향 없이** 새로운 사용자와 애플리케이션을 지원하기 위해 **빠르게(빠르게) 쉽게 확장**할 수 있는 능력

### 어떻게 가능한가?

> 네트워크 설계자들이 **표준(Standards)과 프로토콜(Protocols)** 을 따르기 때문에 확장이 가능하다 (= 표준에 의해 확장)

- 어떤 장비를 추가해도 기존 네트워크와 **호환**됨
- 추가 사용자 및 전체 네트워크를 **기존 사용자 성능 저하 없이(호환적인)** 인터넷에 연결 가능

### 실생활 예시

```
회사 직원 100명 → 500명으로 증가
→ 스위치/라우터 추가해도 기존 직원들 속도 유지
```

## 4. Quality of Service (서비스 품질)

> 음성 통화나 라이브 영상은 **더 높은 품질 보장**이 필요하다.

라이브 영상이 끊기는 이유:
- **사용 가능한 대역폭보다 수요가 더 많을 때**
- **QoS가 설정되지 않았을 때** (설정되어 있지 않기 때문에)

### QoS의 역할

- 모든 사용자에게 **안정적인 콘텐츠 전달을 보장**하는 핵심 메커니즘
- QoS 정책이 적용되면 **라우터가 데이터와 음성 트래픽의 흐름을 더 쉽게 관리**
- 라우터가 **통신 유형과 중요도에 따라 우선순위를 맞춰준다**

### 우선순위 기준

```
높은 우선순위  →  VoIP(인터넷 전화), 화상회의, 라이브 스트리밍
낮은 우선순위  →  웹 페이지 로딩 (웹 페이지는 하순위가 낮다)
```

### QoS vs QoE 비교

| 구분 | QoS (Quality of Service) | QoE (Quality of Experience) |
|------|--------------------------|------------------------------|
| **관점** | 네트워크 장비 (객관적) | 사용자 체감 (주관적) |
| **측정** | 대역폭, 지연시간, 패킷손실 | 만족도, 불편함 여부 |
| **예시** | 초당 몇 Mbps 전송됐는가 | 영상이 끊기지 않고 쾌적했는가 |

## 5. Network Security (네트워크 보안)

### 보안의 2가지 유형

| 유형 | 내용 |
|------|------|
| **Network Infrastructure Security** | 물리적 보안, 허가 받지 못한 사람이 설정을 바꾸게 두지 않기 위한 무단 접근 방지 |
| **Information Security** | 네트워크를 통해 전송되는 **데이터/정보 자체 보호** |

### 보안의 3가지 목표 (CIA Triad)

| 목표 | 이름 | 설명 | 핵심 키워드 |
|------|------|------|-------------|
| **Confidentiality** | 기밀성 | 의도된 수신자만 데이터를 읽을 수 있어야 함 | 지정된 수신자만 |
| **Integrity** | 무결성 | 전송 중 데이터가 변조되지 않았음을 보장 | 전송 중 변경 불가 |
| **Availability** | 가용성 | 권한 있는 사용자가 데이터에 적시에 안정적으로 접근 가능 | 균일 접근·접속 |

### CIA 실생활 예시

| 목표 | 예시 |
|------|------|
| **Confidentiality** | 카카오톡 암호화 → 나와 상대방만 읽을 수 있음 |
| **Integrity** | 인터넷 뱅킹 송금 시 금액이 전송 중에 바뀌지 않음 |
| **Availability** | DDoS 공격을 받아도 정상 고객은 계속 서비스 이용 가능 |

### 그림 설명

```
[Internet]
    ↓ 외부 위협
[방화벽 🧱]   ← 관리자가 소프트웨어·하드웨어 보안 +
    ↓           장치에 대한 물리적 접근 차단으로 네트워크 보호
[내부 네트워크]  ← 보안 조치는 네트워크를 무단 접근으로부터 보호
```

## 전체 개념 요약

```
Reliable Networks
│
├── Network Architecture (4가지 특성)
│   ├── Fault Tolerance  → 장애 발생해도 계속 동작
│   ├── Scalability      → 확장해도 기존 성능 유지
│   ├── QoS              → 중요 트래픽 우선 처리
│   └── Security         → CIA (기밀성·무결성·가용성)
│
├── Fault Tolerance
│   ├── Packet Switching → 패킷마다 다른 경로 가능 (인터넷)
│   └── Circuit Switching → 전용 회선 고정 (옛날 전화)
│
├── Scalability
│   └── 표준·프로토콜 덕분에 장비 추가해도 호환 가능
│
├── QoS
│   ├── 라우터가 트래픽 우선순위 관리
│   ├── Real-time    → 10ms 이내
│   ├── Streaming    → 100ms ~ 170ms
│   └── QoS(수치) vs QoE(체감) 구분
│
└── Security (CIA Triad)
    ├── Confidentiality → 지정된 수신자만
    ├── Integrity       → 전송 중 변조 없음
    └── Availability    → 권한자는 언제든 접근 가능
```

---

## 배운 점

- **Fault Tolerance**는 Packet Switching 방식 덕분에 가능하며, Circuit Switching은 전용 회선이 끊기면 통신 자체가 불가능하다는 근본적 한계가 있다
- **Scalability**는 "무조건 장비만 추가하면 된다"가 아니라, **표준과 프로토콜을 따르기 때문에** 호환성이 보장되어 확장이 가능한 것이다
- **QoS**는 대역폭이 부족할 때 중요한 트래픽(VoIP, 화상회의)을 먼저 처리하게 해주며, Real-time은 10ms, Streaming은 100~170ms 기준을 지켜야 한다
- **QoS(수치)와 QoE(체감)** 는 다를 수 있으며, QoS가 높아도 QoE가 낮을 수 있다
- **보안의 CIA**는 기밀성·무결성·가용성으로, 세 가지 모두 갖춰야 완전한 보안이 된다
