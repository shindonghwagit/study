# Common Types of Networks (네트워크의 일반적인 유형)

---

## 학습 목표

- 네트워크의 **규모별 유형**을 구분하고 각 특징을 설명할 수 있다
- **LAN과 WAN**의 차이점을 비교할 수 있다
- **인터넷**의 구조와 관리 기관을 설명할 수 있다
- **인트라넷과 엑스트라넷**의 차이를 구분할 수 있다

---

## 1. Networks of Many Sizes (다양한 규모의 네트워크)

네트워크는 규모에 따라 크게 4가지로 나뉜다.

| 유형 | 설명 | 예시 |
|------|------|------|
| **Small Home Networks** | 소수의 컴퓨터를 서로 + 인터넷에 연결 | 가정용 공유기에 연결된 PC, 스마트폰 |
| **SOHO** (Small Office/Home Office) | 가정 또는 원격 사무실에서 기업 네트워크에 연결 | 재택근무자의 VPN 연결 |
| **Medium to Large Networks** | 수백~수천 대의 컴퓨터가 여러 위치에 연결 | 대학교, 대기업 사내 네트워크 |
| **World Wide Networks** | 전 세계 수억 대의 컴퓨터를 연결 | 인터넷 |

```
규모 순서
Small Home  <  SOHO  <  Medium/Large  <  World Wide (Internet)
```

---

## 2. LANs and WANs

네트워크 인프라는 다음 요소에 따라 크게 달라진다:

- **커버하는 지역의 크기** (Size of the area covered)
- **연결된 사용자 수** (Number of users connected)
- **제공 서비스의 수와 종류** (Number and types of services available)
- **책임 영역** (Area of responsibility)

가장 대표적인 두 가지 네트워크 유형은 **LAN**과 **WAN**이다.

---

### LAN (Local Area Network) - 근거리 통신망

> **소규모 지역**에 걸쳐 있는 네트워크 인프라

| 특징 | 내용 |
|------|------|
| 연결 대상 | 제한된 구역 내의 End Device들 |
| 관리 주체 | 단일 조직 또는 개인 |
| 속도 | 내부 장치에 고속 대역폭 제공 |

---

### WAN (Wide Area Network) - 광역 통신망

> **넓은 지역**에 걸쳐 있는 네트워크 인프라

| 특징 | 내용 |
|------|------|
| 연결 대상 | 넓은 지역에 걸쳐 LAN들을 연결 |
| 관리 주체 | 하나 이상의 서비스 제공업체(SP) |
| 속도 | LAN 간 연결은 상대적으로 느림 |

### LAN vs WAN 비교

| 구분 | LAN | WAN |
|------|-----|-----|
| **범위** | 건물, 캠퍼스 등 좁은 지역 | 도시, 국가, 대륙 등 넓은 지역 |
| **관리** | 개인 또는 단일 조직 | ISP 등 서비스 제공업체 |
| **속도** | 고속 | 상대적으로 느림 |
| **연결 대상** | End Device들 | LAN들 |

> 그림에서 **빨간 번개선**은 WAN 링크를 나타냄 — 서로 다른 LAN이 WAN을 통해 연결된 것을 표현

---

## 3. The Internet (인터넷)

> **전 세계에 걸쳐 연결된 LAN과 WAN의 집합체**

- LAN들은 **WAN을 통해** 서로 연결됨
- WAN은 **구리선, 광섬유 케이블, 무선 전송** 방식을 사용

### 소유권

인터넷은 **특정 개인이나 단체가 소유하지 않는다.**
대신 아래 기관들이 인터넷 구조 유지를 위해 운영됨:

| 기관 | 풀네임 | 역할 |
|------|--------|------|
| **IETF** | Internet Engineering Task Force | 인터넷 기술 표준(프로토콜) 개발 및 관리 |
| **ICANN** | Internet Corporation for Assigned Names and Numbers | 도메인 이름 및 IP 주소 관리 |
| **IAB** | Internet Architecture Board | 인터넷 아키텍처 전반의 방향성 감독 |

### 인터넷 구조

```
[Home LAN]      ─┐
[Hospital LAN]  ─┤
[Government LAN]─┤── WAN 백본 (라우터들) ── 전 세계 연결
[School LAN]    ─┤
[Business LAN]  ─┘
```

## 4. Intranets and Extranets

### Intranet (인트라넷)

> 조직 **내부에서만** 접근 가능한 사설 LAN/WAN의 집합

- 조직 구성원 또는 **권한을 부여받은 사람만** 접근 가능
- 외부 인터넷과 분리된 **폐쇄적 네트워크**
- 예: 사내 업무 포털, 교내 학사 시스템

### Extranet (엑스트라넷)

> 외부 조직의 특정 인원에게 **제한적으로 접근을 허용**하는 네트워크

- 협력사, 공급업체, 고객사 등에게 **보안 접근** 제공
- 인트라넷보다 열려 있지만, 인터넷처럼 완전 공개는 아님
- 예: 납품업체가 우리 회사 재고 시스템에 접근

### 접근 범위 비교 (동심원 구조)

```
┌─────────────────────────────────┐
│          The Internet           │  ← 전 세계 누구나 (The World)
│   ┌─────────────────────────┐   │
│   │        Extranet         │   │  ← 협력사·공급업체·고객사
│   │  ┌───────────────────┐  │   │
│   │  │     Intranet      │  │   │  ← 내부 직원만 (Company Only)
│   │  └───────────────────┘  │   │
│   └─────────────────────────┘   │
└─────────────────────────────────┘
```

| 구분 | 접근 대상 | 개방 수준 |
|------|-----------|-----------|
| **Intranet** | 내부 직원만 | 완전 폐쇄 |
| **Extranet** | 내부 + 허가된 외부 조직 | 부분 개방 |
| **Internet** | 전 세계 누구나 | 완전 개방 |

```
Common Types of Networks
│
├── 규모별 분류
│   ├── Small Home    → 가정 내 소수 PC
│   ├── SOHO          → 원격 근무 환경
│   ├── Medium/Large  → 수백~수천 대, 다수 거점
│   └── World Wide    → 인터넷
│
├── LAN vs WAN
│   ├── LAN → 좁은 지역, 단일 조직 관리, 고속
│   └── WAN → 넓은 지역, SP 관리, 상대적으로 느림
│
├── The Internet
│   ├── LAN + WAN의 전 세계적 집합체
│   └── 관리 기관: IETF / ICANN / IAB
│
└── Intranet / Extranet
    ├── Intranet → 내부 전용 (Company Only)
    └── Extranet → 허가된 외부 조직까지 허용
```

---

## 배운 점

- 네트워크는 규모에 따라 용도와 구조가 완전히 달라진다. 가정용 공유기부터 인터넷까지 모두 같은 원리의 연장선임을 이해하게 되었다
- **LAN은 내가 직접 관리**, **WAN은 통신사가 관리**한다는 관리 주체의 차이가 핵심이다
- 인터넷은 누군가 소유하는 것이 아니며, IETF·ICANN·IAB 같은 국제 기관들이 표준과 구조를 유지한다
- **Intranet vs Extranet**은 접근 허용 범위의 차이이며, 기업 보안 정책과 직결되는 개념이다
