# 비지도 학습 및 강화 학습

> **Easy 딥러닝** 교재 기반 학습 정리  
> 발표자: 신동화 (순천향대학교)

---

## 학습 목표

- 레이블 없이 데이터의 구조를 파악하는 **비지도 학습**의 핵심 알고리즘을 이해한다
- **군집화**(K-Means, DBSCAN)와 **차원 축소**(PCA)의 동작 원리와 차이점을 설명할 수 있다
- **강화 학습**의 기본 구조(Agent–Environment 상호작용)를 이해한다
- Q-Learning의 학습 과정과 이를 보완하는 **Exploration**, **Discount Factor** 개념을 설명할 수 있다

---

## 1. 비지도 학습 (Unsupervised Learning)

레이블이 없는 데이터에서 패턴이나 구조를 스스로 찾아내는 학습 방식이다. 크게 **군집화**와 **차원 축소** 두 가지 방향으로 나뉜다.

---

### 1.1 군집화 (Clustering)

비슷한 특성을 가진 데이터들을 그룹으로 묶는 방법이다.

#### K-Means Clustering

**거리 기반** 군집화 알고리즘으로, 데이터를 K개의 군집으로 나눈다.

**알고리즘 동작 과정**

1. 군집 수 K를 정하고, K개의 **중심점(centroid)** 을 임의로 배치한다
2. 각 샘플에 대해 가장 가까운 centroid를 매핑한다
3. 매핑된 샘플들의 평균 위치로 centroid를 이동한다
4. 매핑이 변하지 않을 때까지 2~3을 반복한다

**거리 측정 — 유클리드 거리(Euclidean Distance)**

$$
d(x, y) = \sqrt{\sum_{i=1}^{n}(x_i - y_i)^2}
$$

각 샘플과 centroid 사이의 유클리드 거리를 계산하여 가장 가까운 centroid에 할당한다.

**특징**
- 군집 수 K를 사전에 지정해야 한다
- 초기 centroid 위치에 따라 결과가 달라질 수 있다
- 구형(spherical) 군집에 적합하다

---

#### DBSCAN (Density-Based Spatial Clustering)

**밀도 기반** 군집화 알고리즘으로, 데이터가 밀집된 영역을 군집으로 판단한다.

**파라미터**

| 파라미터 | 설명 |
|---------|------|
| **ε (Epsilon)** | 이웃을 정의하기 위한 반경 |
| **minPts** | 군집을 형성하기 위해 필요한 최소 점의 개수 |

**핵심 개념 — 점의 분류**

| 유형 | 정의 |
|------|------|
| **Core point** | ε 반경 내에 minPts 이상의 점을 가진 중심점 |
| **Border point** | Core point의 ε 반경 내에 있지만, 자신의 ε 반경에는 minPts 미만의 점을 가진 점 |
| **Noise point** | Core point에도 Border point에도 속하지 않는 점 (이상치) |

**K-Means와 비교**

| 항목 | K-Means | DBSCAN |
|------|---------|--------|
| 기반 | 거리 (centroid) | 밀도 (ε, minPts) |
| 군집 수 | 사전 지정 필요 (K) | 자동 결정 |
| 이상치 처리 | 불가 | Noise point로 분리 가능 |
| 군집 형태 | 구형에 적합 | 임의 형태 가능 |

---

### 1.2 차원 축소 (Dimension Reduction)

데이터의 중요한 특성은 유지하면서 데이터의 복잡성(차원)을 줄이는 방법이다.

#### PCA (Principal Component Analysis, 주성분 분석)

고차원 데이터에서 **분산이 가장 큰 방향**(주성분)을 찾아 낮은 차원으로 투영하는 기법이다.

**동작 원리**

1. 데이터 분포의 **평균 지점**을 구한다
2. 데이터의 분포를 가장 잘 설명할 수 있는 **주성분 벡터**(분산이 최대인 방향)를 찾는다
3. 데이터를 해당 벡터에 **정사영(orthogonal projection)** 하여 차원을 축소한다

**수식 — 공분산 행렬의 고유값 분해**

$$
C = \frac{1}{n-1} X^T X
$$

공분산 행렬 $C$의 고유벡터(eigenvector)가 주성분 방향이 되고, 대응하는 고유값(eigenvalue)이 해당 방향의 분산 크기를 나타낸다. 고유값이 큰 순서대로 주성분을 선택하여 차원을 축소한다.

## 2. 강화 학습 (Reinforcement Learning)

환경과 상호작용하며 **보상을 최대화하는 방향**으로 에이전트가 스스로 학습하는 방식이다.

**기본 구성 요소**

| 요소 | 설명 |
|------|------|
| **Agent** | 행동을 취하는 주체 |
| **Action** | Agent가 취할 수 있는 모든 행동 |
| **Reward** | Agent가 취한 Action에 대한 보상 |
| **Environment** | 강화 학습이 일어나는 무대 (환경) |
| **State** | 환경의 현재 상태 |

Agent는 Environment 속에서 State를 인식 → Action을 선택 → Reward를 받는 순환을 반복하며 학습한다.

---

### 2.1 Q-Learning

**Q-map**(상태-행동 가치 테이블)을 경험을 통해 학습하고, 이를 활용해 **최적의 행동(Greedy action)** 을 선택하는 알고리즘이다.

**Q-value 업데이트 수식 (Bellman Equation 기반)**

$$
Q(s, a) \leftarrow Q(s, a) + \alpha \left[ R + \gamma \max_{a'} Q(s', a') - Q(s, a) \right]
$$

| 기호 | 의미 |
|------|------|
| $Q(s, a)$ | 상태 $s$에서 행동 $a$를 취했을 때의 가치 |
| $\alpha$ | 학습률 (learning rate) |
| $R$ | 받은 보상 |
| $\gamma$ | 할인 계수 (discount factor) |
| $s'$ | 행동 후 도달한 다음 상태 |

**한계**: 한번 찾은 최적 경로만 반복하여, 더 좋은 경로가 있어도 탐색하지 못한다.

---

### 2.2 Exploration vs Exploitation

Q-Learning의 한계를 보완하기 위해 **새로운 경로 탐색(Exploration)** 과 **기존 지식 활용(Exploitation)** 의 균형이 필요하다.

**ε-Greedy 알고리즘**

$$
a = \begin{cases} \text{random action} & \text{확률 } \varepsilon \\ \arg\max_a Q(s, a) & \text{확률 } 1 - \varepsilon \end{cases}
$$

| 전략 | 설명 |
|------|------|
| **Exploitation** | 학습된 Q-map에서 최선의 행동만 선택 |
| **Exploration** | 무작위로 새로운 행동을 시도하여 미지의 경로 탐색 |

**ε 조절 전략**: 초기에는 ε을 높게(예: 0.9) 설정하여 탐색을 많이 하고, 학습이 진행될수록 ε을 점차 줄여서 학습된 정책을 활용한다.

---

### 2.3 Discount Factor (γ)

여러 경로를 발견했을 때, **어떤 경로가 더 효율적인지** 판단하기 위해 미래 보상을 현재 가치로 환산하는 비율이다.

$$
G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k R_{t+k+1}
$$

| γ 값 | 의미 |
|------|------|
| γ → 1 | 미래 보상을 거의 그대로 반영 (장기적 관점) |
| γ → 0 | 즉각적인 보상만 중시 (근시안적) |

**핵심 원리**: 목표 지점에서 1칸 떨어진 곳은 γ, 2칸은 γ², 3칸은 γ³의 가치를 갖는다. 멀리 있을수록 가치가 감소하므로 **짧은 경로가 자연스럽게 높은 가치**를 갖게 되어 효율적인 경로를 찾을 수 있다.

---

## 배운 점

- **비지도 학습**은 레이블 없이도 데이터의 구조를 파악할 수 있으며, 군집화와 차원 축소라는 서로 다른 접근법이 상호 보완적으로 활용된다
- K-Means는 직관적이고 구현이 간단하지만, DBSCAN은 **이상치 처리**와 **비정형 군집** 탐지에서 강점을 보인다
- 강화 학습은 단순한 Q-Learning만으로는 부족하며, **Exploration–Exploitation 균형**(ε-Greedy)과 **Discount Factor**(γ)를 통해 점진적으로 보완해야 한다는 것을 알게 되었다
- 각 알고리즘이 어떤 문제를 해결하기 위해 등장했는지, **"왜 이 방법이 필요한가"** 의 흐름을 따라가니 개념이 훨씬 명확하게 이해되었다
